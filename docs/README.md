# `docs/`

Project documentation, organized by the Keel taxonomy:

- `architecture/` — system design and layering (`ARCHITECTURE.md` today).
- `adr/` — architecture decision records (e.g. adopting the Keel structure).
- `specs/` · `design/` · `guides/` · `reference/` — added as content lands.

Every Markdown file here carries frontmatter (see [`CONVENTIONS.md`](../CONVENTIONS.md)).
The seven-steps deep-dive is generated to `SEVEN_STEPS.md` from the web content
model and also rendered in the wiki.
